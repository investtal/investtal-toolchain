# 9cc — Docker Sandbox Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: use `subagent-parallel-execution` (recommended) or inline execution via `finishing-execution` to implement task-by-task. Steps use `- [ ]` checkboxes. Every code task follows **RED → GREEN → REFACTOR** (TDD): write failing test, run it (fails), write minimal code, run it (passes), refactor, commit.

**Goal:** Add a `--sandbox` flag to `9cc run` that runs Claude Code inside a Docker container, isolating the host home directory, SSH, AWS, and shell while keeping the workspace and Claude Code auth/settings accessible. Outbound Claude tool/prompt requests are logged via an in-container agent-proxy. Normal `9cc run` stays host-native.

**Architecture:** Introduce a `9cc sandbox` subcommand family (`9cc sandbox build`) and a `--sandbox` flag on `9cc run`. A `9cc/Dockerfile` builds an image from `node:24-slim` from scratch, installing proto, dev tools, zsh, and copying the host's already-installed Claude Code binary (`~/.claude/local`) and team context (`~/.claude`, `~/.investtal`, `~/.proto`, `~/.prototools`). An embedded `agent-proxy.mjs` runs as a sidecar inside the container; it forwards Claude Code to the real upstream API and writes Markdown request logs to a host-mounted egress directory. The bash and PowerShell launchers remain the only entry points; no new top-level binaries are installed.

**Tech Stack:** Bash 3.2+/4+, PowerShell 5.1+, Docker, Node.js (existing JSON parsing, plus agent-proxy). No additional runtimes. Tests stub `docker` and `claude`.

## Global Constraints

- **Default mode is host-native:** `9cc run <model>` runs `claude` on the host as before.
- **Sandbox mode via flag:** `9cc run --sandbox <model>` launches the sandbox.
- **Docker only:** Podman and other container runtimes are out of scope.
- **Image built by 9cc from scratch:** `9cc/Dockerfile` is the source of truth, built on demand with `9cc sandbox build`. Base image `node:24-slim`.
- **Claude CLI from host:** Copy the host's already-installed Claude Code binary at `~/.claude/local` into the image (not npm install). If not present, error cleanly.
- **No host secret mounts:** `~/.claude`, `~/.investtal`, `~/.proto`, `~/.prototools` are copied into the image at build time; they are not bind-mounted at runtime.
- **Runtime read-only auth:** Only `~/.claude/settings.json` is bind-mounted read-only for auth. The rest of `~/.claude` is already inside the image.
- **Workspace mount:** Current working directory is bind-mounted as the workspace.
- **Home/root guard:** Refuse to run if CWD is `~` or `/`.
- **Isolated env:** Pass only a minimal allowlist of env vars (`ANTHROPIC_*`, `CLAUDE_*`, `CC9_*`, `TERM`, `HOME`, `USER`, `PATH` from container) into the container. Do not pass the entire host shell environment.
- **Egress logging:** Run agent-proxy inside the container. Claude Code talks to `http://localhost:8787`, proxy forwards to the real `ANTHROPIC_BASE_URL` read from settings, and logs every tool/prompt request to Markdown in a host-mounted egress directory.
- **Zsh included:** Install `zsh` and copy the host `~/.zshrc` and `~/.zsh_*` config into the image if present.
- **Cross-platform:** Bash and PowerShell launchers must both implement the same behavior.
- **No config mutation:** Never write to `~/.claude/settings.json`.
- **TDD:** Every new function gets a failing test first. `docker` and `claude` are never invoked in tests (stub them).

   ```bash
   #!/usr/bin/env bash
   # Container entrypoint: start agent-proxy, then run claude through it.
   set -euo pipefail

   : "${AGENT_PROXY_PORT:=8787}"
   : "${ANTHROPIC_BASE_URL:=https://api.anthropic.com}"
   : "${AGENT_PROXY_LOG_DIR:=/tmp/9cc-egress}"

   mkdir -p "$AGENT_PROXY_LOG_DIR"

   # Start proxy in background, forwarding to the real upstream.
   UPSTREAM_URL="$ANTHROPIC_BASE_URL" PORT="$AGENT_PROXY_PORT" LOG_DIR="$AGENT_PROXY_LOG_DIR" \
       node /home/9cc/agent-proxy.mjs &
   PROXY_PID=$!

   # Wait for proxy to listen.
   for _ in $(seq 1 30); do
       if curl -fsS "http://localhost:$AGENT_PROXY_PORT" >/dev/null 2>&1; then break; fi
       sleep 0.1
   done

   # Point Claude at the proxy, leaving the rest of the env allowlist to the launcher.
   export ANTHROPIC_BASE_URL="http://localhost:$AGENT_PROXY_PORT"
   export PATH="/home/9cc/.claude/local/bin:$PATH"

   cleanup() { kill "$PROXY_PID" 2>/dev/null || true; }
   trap cleanup EXIT

   exec claude "$@"
   ```

   ```bash
   #!/usr/bin/env bash
   # 9cc sandbox helpers — isolate Claude Code in Docker.
   set -euo pipefail

   : "${CC9_HOME:=${HOME}/.9cc}"
   : "${CLAUDE_SETTINGS:=${HOME}/.claude/settings.json}"
   : "${CC9_SANDBOX_IMAGE:=9cc-sandbox:latest}"
   : "${CC9_SANDBOX_CONTEXT:=${CC9_HOME}/sandbox-context}"

   is_guarded_dir() {
       local cwd; cwd="$(cd "$1" && pwd -P)"
       local home; home="$(cd "$HOME" && pwd -P)"
       if [ "$cwd" = "$home" ] || [ "$cwd" = "/" ]; then
           echo "9cc sandbox: refusing to run in $cwd (mount would expose home or root)" >&2
           return 1
       fi
   }

   _copy_if_exists() {
       local src="$1" dst="$2"
       if [ -e "$src" ]; then cp -a "$src" "$dst"; fi
   }

   _copy_dir_if_exists() {
       local src="$1" dst="$2"
       if [ -d "$src" ]; then cp -a "$src" "$dst"; fi
   }

   build_image() {
       command -v docker >/dev/null 2>&1 || { echo "9cc sandbox: docker not found" >&2; return 1; }
       local claude_local="$HOME/.claude/local"
       [ -d "$claude_local" ] || { echo "9cc sandbox: $claude_local not found; install Claude Code first" >&2; return 1; }

       local ctx="$CC9_SANDBOX_CONTEXT"
       rm -rf "$ctx"
       mkdir -p "$ctx"

       _copy_dir_if_exists "$claude_local" "$ctx/claude-local"
       _copy_dir_if_exists "$HOME/.claude" "$ctx/claude"
       _copy_dir_if_exists "$HOME/.investtal" "$ctx/investtal"
       _copy_dir_if_exists "$HOME/.proto" "$ctx/proto"
       _copy_if_exists "$HOME/.prototools" "$ctx/prototools"
       _copy_if_exists "$HOME/.zshrc" "$ctx/zshrc"
       _copy_if_exists "$HOME/.zshenv" "$ctx/zshenv"
       [ -f "$ctx/zshrc" ] || touch "$ctx/zshrc"
       [ -f "$ctx/zshenv" ] || touch "$ctx/zshenv"

       local this_dir; this_dir="$(dirname "$(readlink -f "${BASH_SOURCE[0]}")")"
       cp "$this_dir/agent-proxy.mjs" "$ctx/agent-proxy.mjs"
       cp "$this_dir/sandbox-entrypoint.sh" "$ctx/sandbox-entrypoint.sh"

       local df; df="$this_dir/Dockerfile"
       [ -f "$df" ] || { echo "9cc sandbox: Dockerfile not found at $df" >&2; return 1; }

       docker build \
           -t "$CC9_SANDBOX_IMAGE" \
           -f "$df" \
           "$ctx"
   }

   egress_dir() {
       local dir; dir="${CC9_HOME}/egress"
       mkdir -p "$dir"
       printf '%s' "$dir"
   }

   run_sandboxed() {
       command -v docker >/dev/null 2>&1 || { echo "9cc sandbox: docker not found" >&2; return 1; }
       local cwd; cwd="$(pwd -P)"
       is_guarded_dir "$cwd" || return 1

       if [ "${CC9_SANDBOX_NO_BUILD:-}" != "1" ]; then
           docker image inspect "$CC9_SANDBOX_IMAGE" >/dev/null 2>&1 || build_image || return 1
       fi

       [ -f "$CLAUDE_SETTINGS" ] || { echo "9cc sandbox: $CLAUDE_SETTINGS not found" >&2; return 1; }

       local egress; egress="$(egress_dir)"
       echo "9cc sandbox: egress logs -> $egress" >&2

       docker run --rm -it \
           --user "$(id -u):$(id -g)" \
           --workdir /workspace \
           -v "$cwd:/workspace" \
           -v "$CLAUDE_SETTINGS:/home/9cc/.claude/settings.json:ro" \
           -v "$egress:/tmp/9cc-egress" \
           -e HOME=/home/9cc \
           -e USER="$(id -un)" \
           -e TERM="${TERM:-xterm-256color}" \
           -e ANTHROPIC_MODEL \
           -e ANTHROPIC_DEFAULT_OPUS_MODEL \
           -e ANTHROPIC_DEFAULT_SONNET_MODEL \
           -e ANTHROPIC_DEFAULT_HAIKU_MODEL \
           -e CLAUDE_CODE_AUTO_COMPACT_WINDOW \
           -e ANTHROPIC_BASE_URL \
           -e ANTHROPIC_API_KEY \
           "$CC9_SANDBOX_IMAGE" \
           claude "$@"
   }
   ```

## Open / Deferred

- **Windows Docker Desktop networking:** The proxy runs inside the main container. This should work on Docker Desktop, but the TTY/interactive `docker run -it` path must be verified.
- **Image size:** `node:24-slim` + `zsh` + `gh` + `proto` is large. Optimize later if builds become slow.
- **Layer caching:** Every build copies the current host context. This is intentional for security. Add a `--no-cache` option later if needed.
- **zsh plugins:** If `~/.zshrc` sources plugins from non-standard paths, they may break inside the container. Document that only self-contained zsh configs are supported.
- **Payload logging:** agent-proxy logs request/response bodies and headers (auth redacted). It does not perform TLS interception; it only sees what Claude Code sends to it, which is sufficient because Claude Code is configured to route through the proxy.
- **Host claude-local staleness:** If the host updates Claude Code, the sandbox image must be rebuilt with `9cc sandbox build` to pick up the new binary.
